New Relic のモデル パフォーマンス監視ソリューションは、ML モデルの監視を容易にする Python ライブラリを提供します。**Bring Your Own Data (BYOD)**を使用すると、モデルの推論とメトリクス データを、ノートブックまたはその他の環境から直接、New Relic プラットフォームにすばやく送信できます。

主な機能:

* 統合が容易

* ML フレームワークに依存しない

* New Relic オブザーバビリティ フレームワークとの完全な互換性

* `newrelic-telemetry-sdk-python`ライブラリに基づく

* モニタリングとアラート:

  * 予測
  * 特徴
  * 欠損値
  * モデルとデータのドリフト
  * カスタム推論データ
  * カスタム パフォーマンス指標

[`newrelic-telemetry-sdk-python`](https://github.com/newrelic/newrelic-telemetry-sdk-python)に基づく[`ml-performance-monitoring`](https://github.com/newrelic-experimental/ml-performance-monitoring) python パッケージを使用すると、コードに数行を追加するだけで、モデルの特徴と予測値、およびカスタム メトリックを送信できます。

python パッケージを使用して、以下の種類のデータを New Relic に送信します。

1. **推論データ:**モデルの特徴と予測値をストリーミングします。推論データは、 `InferenceData`という名前の**[カスタム イベント](/docs/data-apis/ingest-apis/introduction-event-api/)**としてストリーミングされます。
2. **データ メトリック:**すべての生の推論データを送信する代わりに、機能と予測に対して集計統計オプションを選択します (たとえば、 `min` 、 `max` 、 `average` 、または`percentile` )。それらは**[メトリクス](/docs/data-apis/ingest-apis/metric-api/introduction-metric-api/)**として自動的に送信されます。
3. **カスタム指標:**独自の指標を計算し、モデルのパフォーマンスまたはモデル データを監視して、 `record_metrics`関数を使用してそれらを New Relic にストリーミングします。これらは**[メトリック](/docs/data-apis/ingest-apis/metric-api/introduction-metric-api/)**として送信されます。
