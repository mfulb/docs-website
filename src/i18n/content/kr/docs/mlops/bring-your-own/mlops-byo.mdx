New Relic의 모델 성능 모니터링 솔루션은 ML 모델을 쉽게 모니터링할 수 있는 Python 라이브러리를 제공합니다. **BYOD(Bring Your Own Data)** 를 사용하면 모델의 추론 및 지표 데이터를 노트북이나 기타 환경에서 직접 New Relic 플랫폼으로 신속하게 보낼 수 있습니다.

주요 기능:

* 쉬운 통합

* ML 프레임워크 불가지론

* New Relic 가관측성 프레임워크와의 완벽한 호환성

* `newrelic-telemetry-sdk-python` 라이브러리 기반

* 모니터링 및 알림:

  * 예측
  * 특징
  * 누락된 값
  * 모델 및 데이터 드리프트
  * 사용자 지정 추론 데이터
  * 사용자 지정 성능 지표

[`newrelic-telemetry-sdk-python`](https://github.com/newrelic/newrelic-telemetry-sdk-python) 를 기반으로 하는 [`ml-performance-monitoring`](https://github.com/newrelic-experimental/ml-performance-monitoring) Python 패키지를 사용하면 코드에 몇 줄만 추가하면 모델의 기능과 예측 값은 물론 맞춤 측정항목을 보낼 수 있습니다.

Python 패키지를 사용하여 다음 유형의 데이터를 New Relic에 보냅니다.

1. **추론 데이터:** 모델의 기능 및 예측 값을 스트리밍합니다. 추론 데이터는 `InferenceData` 라는 **[맞춤 이벤트](/docs/data-apis/ingest-apis/introduction-event-api/)** 로 스트리밍됩니다.
2. **데이터 메트릭:** 모든 원시 추론 데이터를 보내는 대신 기능 및 예측에 대해 집계된 통계 옵션을 선택합니다(예: `min` , `max` , `average` 또는 `percentile` ). **[측정항목](/docs/data-apis/ingest-apis/metric-api/introduction-metric-api/)** 으로 자동 전송됩니다.
3. **맞춤 측정항목:** 자체 측정항목을 계산하고 모델 성능 또는 모델 데이터를 모니터링하고 `record_metrics` 함수를 사용하여 New Relic으로 스트리밍합니다. **[측정항목](/docs/data-apis/ingest-apis/metric-api/introduction-metric-api/)** 으로 전송됩니다.

## 추론 데이터와 메트릭을 New Relic으로 스트리밍하려면 어떻게 해야 합니까? [#stream-data]

이 [빠른 시작](https://github.com/newrelic-experimental/ml-performance-monitoring) 을 확인하십시오
